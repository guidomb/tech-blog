XCode 6.2 Broke Objective-C’s Semantics

Mar 21st, 2015 4:01 pm | Comments

Yeah the new “stable” version of XCode is out!!! Version 6.2, we can start hacking for the Apple Watch. The problem is that after installing XCode 6.2 the project I am working on stopped compiling. Without even reading the error message I did the usual thing when a project stops compiling out of the sudden.

  • cmd + option + shift + k
  • sudo rm -frd /var/folders/*
  • rm -frd ~/Library/Developers/Xcode/DerivedData/*

I did not work so it was time to read the error message:

Use of undeclared identifier '_deviceManager'

The piece of code that was causing the problem was a simple override of a read only property getter method:

The actual code in the previous snippet does not matter, it is just lazily initializing an instance variable inside the getter method. What matters is that that piece of code was compiling before I installed XCode 6.2.

After a while I realized that if you declare a property and then you override a getter or a setter the implicit instance variable generated by the compiler disappears. I solved this issue by adding @synthesize deviceService = _deviceService;.

What bothers me is that this was something that was working and they broke it in a minor update of the IDE. What is going on with Apple’s Q&A? Apple’s software does not seem to be as good and as it used to be.

The release notes do not mention anything related to changing the behavior of properties so I assume that this is a bug. I submitted the issue to Apple and opened a radar. In the meantime I am fixing code that used to compile.

Thanks Apple.
